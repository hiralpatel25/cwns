Assigning Local Users

It is the State User Manager’s responsibility to confirm the identity and credentials of the Secondary State or Local User who is given access to the DEP.  In order for a Local User to provide feedback to the State via the DEP, two steps must be taken:

 

1. A Local User account must be approved by the State User Manger, so the Local User can enter the DEP.  The Local User must be granted appropriate access levels (e.g., Facility Update- Feedback, Facility Update- Partial, Submit for State Review).  The State can restrict which facilities/projects the Local User can access (Chapter 6).

 

2. A feedback copy of each facility/project to be updated must be generated.  In the Review Status portlet, the Feedback Status is changed from blank to Local in Progress to generate the feedback copy.  Multiple facilities/projects can be assigned Local Assigned status using the link Local Assigned in the Facility/Project List (Chapter 5).  .

 

If the facility/project has any of the following statuses, State Assigned, State In Progress or State Correction Requested, and a Feedback copy exists with status of Local Assigned or Local in Progress, the  warning message “A Feedback version already exists for this facility. Any changes will not be reflected on the Feedback version” is displayed in all the data areas.

 

After logging in to the DEP, Local Users can request access to additional facilities.  Their requests are shown in the Announcements portlet on the CWNS 2008 Welcome page.
