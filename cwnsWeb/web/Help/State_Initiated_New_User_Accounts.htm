State Initiated New User Accounts

To create a new user account, first perform a search in the User Search portlet to see if the user already has an existing CWNS user account.  If the user is found, then select the user and note the CWNS user Id for the steps below.

 

After clicking the Add User hyperlink from the User List, the User Details portlet displays the area Preliminary User Information which asks a series of question about the new user (Figure 6-3).

 

 

Figure 6- 3: State Initiated New User Account Preliminary Information Screen

 

 

 

• Indicate whether or not the user has an existing CWNS account by selecting the radio button for either yes or no.  Select yes for all people who had a CWNS account in 2004 and provide their CWNS user id.  If yes is selected and a user id is provided, the question “Type of role this user is expected to perform?” is not asked.  The role is assigned based on the user’s previous role and access levels.

 

• Indicate whether or not the user has an existing portal account by selecting the radio button for either yes or no.  If yes is selected, provide the user’s Portal UserId.  If yes is selected, the User Profile Information is read-only.

 

• Select the user’s expected Role by selecting the radio button for either State User or Local User.  Only one role can be selected in the Preliminary User Information area.  Additional Roles can be added after the user is associated with the State.

 

• Select to continue the New User creation process or to end the New User creation process.

 

The User Profile Information area  (Figure 6-4) requests the information needed for establishing the new user’s portal account.  For user’s with existing portal accounts, this screen is view only.  Data fields are explained in Table 6-3; all data fields are required as indicated by the red asterisk (*).

• Select to continue the New User creation process or to return to the Preliminary User Information.

 

The State User Manager is redirected to the User Detail page and must approve the new user to complete the process (see New User Approval). At this screen, changes can be made to the user’s Role(s), Access Level(s), and Facility(ies)/Project(s) (see Editing User Details).
