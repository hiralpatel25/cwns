User Initiated New User Accounts

Potential Local and Secondary State Users can apply for a CWNS user account at http://portal.epa.gov/CWNS or http://www.epa.gov/cwns/cwns2008.htm.

 

The potential user follows the following actions:

 

• The potential user clicks the hyperlink Click here to apply to begin the application process and is redirected to the Preliminary User Information screen (Figure 6-5).  The data fields are described in Table 6-4; all data fields are required.

 

 

Table 6- 5: Explanations of Preliminary User Profile Information Data Fields

 

Data Field

Yes

Do you have an existing EPA Portal Account?

Clicks the radio button for either yes or no to indicate whether or not he/she has an existing portal account. If yes, enters Portal UserId (e.g. john_doe).

Yes

Indicate the State in which the Facility(ies)/ Project(s) are located.

Select the appropriate state from the dropdown list.  

Yes

List the names of the Facility/Project of interest.

Uses the text boxes to enter the name(s) of the facility(ies)/project(s).

Yes

List the names of the town(s) or county(ies) of the Facility/Project of interest.

Use the text box to provide more specific location information (town or county) for the requested facility(ies)/project(s).

Yes

Affiliation

Indicates his/her employer’s name or the organization that he/she is representing.

Yes

Yes

What type of Role will you perform?

Selects the radio button that best describes the type of role he/she will have for CWNS 2008 (see Table 6-1for definitions of roles and access levels).  Only State or Local (not both) can be selected in the Preliminary User Information area.  Additional Roles can be added by the State User Manager after the applicant is accepted as a CWNS user.  

If Local User is selected, indicates how  he/she is interested in participating by checking one or more of the following: Provide feedback on survey data (Facility Update-Feedback), Add/Update Utility Management and Unit Process Information (Facility Update-Partial); Manage other Local User Accounts (CWNS Local User Management).

• Selects to continue the New User creation process or to end the New User creation process.

• On the User Profile Information screen (Figure 6-6), he/she provides the information needed for requesting a portal account.  Data fields are explained in Table 6-6; all data fields are required as indicated by the red asterisk (*).

 

 

Table 6- 6: Explanations of User Details Data Fields

 

Data Field

Explanation

User Details

CWNS User ID

Unique identifier generated for CWNS users by the DEP.  This field is view only.

Name

This user’s name in the format “last name, first name.”  This field is view only.

Status

Select Active from the dropdown list to permit the user to have access to the DEP.  Chose Inactive from the dropdown list to deny the user access to the DEP

Email

The user’s email address.  Enter or edit the user’s email address.

Phone Number

The user’s phone number.  Enter or edit the user’s ten-digit telephone.

Roles

Primary

If there is more than one role, primary indicates which view is shown on the CWNS Welcome Page when the user logins.  Select the radio button of the desired Primary Role.  The change takes place the next time that the user logins.

Role

Indicates the user’s Role(s) (e.g., Federal, State, Local)

Access

Indicates the user’s Access Levels.  See Table 6-1 for an explanation of Access Levels.

After successfully completing the request for access, the potential user is directed to a Thank You screen.  He/she is also sent a confirmation e-mail from cwns@epa.gov indicating: “Your request for access to the Clean Watersheds Needs Survey system has been received. You will be contacted by your State CWNS personnel within five business days.”

 

The State Manager must approve the new user to complete the process.
