Cost Curve Population

The Cost Curve data area is only enabled if the facility has a Cost Curve that requires Cost Curve population.  The DEP prompts the user to enter the data needed to successfully run the Cost Curve (Figures 10-4, 10-5).  Table 10-3 provides additional information on the Cost Curve population data requirements for individual Cost Curves.  The message “No Cost Curve assigned to Facility” is displayed when the data area is not required for the facility

 

 

Table 10- 3: Explanation of Cost Curve Population Data Fields  

 

Cost Curve

Explanation

New Separate Sewer System

Enter a value that is less than or equal to the facility’s [Projected Receiving Collection (residential + non-residential) – Present Receiving Collection (residential + non-residential)].

 

The population must be less than 15,000 persons.

Rehab/Replace Separate Sewer System Population

Enter a value less than or equal to the facility’s Present Population Receiving Collection (residential + non-residential).

 

The population must be less than 15,000 persons.

New Onsite Wastewater Treatment System – all

Enter a value for the number of either Residential or Non-residential (or both) units.

 

The value must less than or equal to the Facility’s Projected Number of Onsite Wastewater Treatment System Units.  If the Number of Units for New Onsite Wastewater Treatment System–all is more than the (Facility’s Projected – Present Number of Onsite Wastewater Treatment System Units), provide a Population Comment explaining the reason.

 

For example, in the following situation it is acceptable for the Cost Curve population for new systems to be greater than the Facility’s Projected Number of Onsite Wastewater Treatment System Units.  The community’s present population is 1000 persons. The projected population includes a new development increasing the population by 200 persons and a decline in the old part of town decreasing the population by 100 persons.  The net population increase is 100 persons (to a future pop of 1,100).  ).  In this case, it is acceptable for the Cost Curve population to be 200 (greater than 1,100 – 1,000).  The situation needs to be outlined in the Population Comment area.

New Onsite Wastewater Treatment System – innovative

Enter a value for the number of either Residential or Non-residential (or both) innovative units.

New Onsite Wastewater Treatment System – conventional

Enter a value for the number of either Residential or Non-residential (or both) conventional units.

 

The number of units of (innovative + conventional) New Onsite Wastewater Treatment System must be less or equal to the Facility’s Residential Projected Number of Onsite Wastewater Treatment System Units.

Rehab Onsite Wastewater Treatment System – all

Enter a value for the number of either Residential or Non-residential (or both) onsite units.

 

The Number of Units for Rehab Onsite Wastewater Treatment System – all must be less or equal to the Facility’s Present Number of Onsite Wastewater Treatment System Units.

Rehab Onsite Wastewater Treatment System – innovative

Enter a value for the number of either Residential or Non-residential (or both) innovative units.

Rehab Onsite Wastewater Treatment System – conventional

Enter a value for the number of either Residential or Non-residential (or both) conventional units.

 

The sum of the Number of Units for Rehab Onsite Wastewater Treatment System – Innovative and Conventional must be less than or equal to the Facility’s non-residential Present Number of Onsite Wastewater Treatment System.

New Cluster System

Enter a value for the number of either Residential or Non-residential (or both) new clustered units.

 

The Number of Units for Cluster Systems must be less than or equal to the Facility’s Projected Number of Cluster System Units.  If the Number of Units for New Cluster System is more than the (Facility’s Projected - Present Number of Cluster System Units), provide a Population Comment.

Rehab Cluster System

Enter a value for the number of either Residential or Non-residential (or both) rehab clustered units.

 

The Number of Units for Rehab Cluster System must be less or equal to the Facility’s Present Number of Cluster System Units.

 

 

As soon as all CCV errors are corrected and saved, the Cost Curve is run.  A check mark in the Cost Allocated column of the Cost Curve data area in Needs indicates that the Cost Curve has been run successfully.  

 

Click to save all changes to this data area.  This will perform all error checking.  Error messages will appear if entered data does not meet all the business rules.  Click to return to the most recently saved version of the data.

 

Warning: If the user does not click before leaving this data area, entered data is not saved.
