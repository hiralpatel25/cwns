Point of Contact

The data area Point of Contact allows users to view, edit, and update the Responsible Entity and other Points of Contact for the facility/project.  Existing Points of Contact data are listed in alphabetical order by Authority.  Point of Contact data from the NPDES permit is displayed if “Use Data” is selected for a NPDES permit in the Permits data area.  The user can edit the Point of Contact data populated from a NPDES permit (Figure 7-7).

 

Note:  When the facility/project data is view-only, a column “Details” is shown in the data table.  Click the to view all the data fields listed in Table 7-5.

 

A check in the column Envirofacts indicates that the source of the Point of Contact data is a NPDES permit. This occurs if “Use Data” is selected for a NPDES permit in the Permits data area.  The user can edit the Point of Contact data populated from a NPDES permit.  If the Point of Contact data is edited, then the checkmark is deleted from the Envirofacts column (Figure 7-7).

 

Remember: A warning message and the Feedback Status “Local Assigned” indicate that a Feedback copy exists for this facility/project.  When a Feedback copy exists, any changes made to the data area are reflected on the Feedback copy.  If a feedback copy has been returned (Feedback Status “State Review Requested”), this symbol indicates that a Feedback Fact Sheet includes changes to this data area.  Click it to view a pop-up box containing Feedback Fact Sheet.

• Click Add Point of Contact to add a new Point of Contact to the facility/project.  Clickin the Edit column to edit an existing Point of Contact.  Both actions open the Add/Edit Point of Contact data area (Figure 7-7).  Explanations of the fields in this data area are in Table 7-5. Required fields are indicated by a red asterisk (*).

 

• Click in the Delete column to remove a Point of Contact from the list.
