New User Approval

It is the User Manager’s responsibility to confirm the identity and credentials of the State Secondary or State User who is given access to the DEP (see User Management Policy).  A user with User Management access can either approve or deny the users application.

 

For State initiated new user accounts, the User Manager is redirected to the User Details portlet to approve or deny user access.   For user initiated new accounts, the applicants are listed in the User List as Pending.

 

• Click on the hyperlinked user name from the User List, and the User Details portlet displays the information provided by the user during self-registration (Figure 6-7, Table 6-5).  

After confirming the user’s identity, the user must be approved to be granted access to the DEP.  Changes can be made to the user’s Role(s), Access Level(s), and Facility(ies)/Project(s) at this time (see Editing User Details).  By default, new users are given the role entered during registration and View access level for all facilities/projects in their own State.

 

• Click to change the user’s status to Active.  The user receives an email from the account cwns@epa.gov to indicate that have been accepted and to provide access information (e.g., User ID, Password).

 

To add additional information to the auto-generated e-mail, add text to the box Registration Email Notification Comments before approving the user.

 

 

• Click to reject the user’s application for access to the DEP.  The user receives an email from the account cwns@epa.gov to indicate that have been denied.

 

To add additional information to the auto-generated e-mail, add text to the box Registration Email Notification Comments before denying the user.
