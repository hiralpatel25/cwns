User Administration Policy

The “delegated” work flow, available only outside of the prior data entry system, is now available in the CWNS DEP via the User Management data area.  In the “delegated” work flow, EPA knows the State CWNS Coordinators and trusts them with delegation rights, specific roles and responsibilities.  In turn, State CWNS Coordinators know the Secondary State and Local CWNS Users; it is up to the State CWNS Coordinators to assign delegation rights, roles, and responsibilities to these users.  Roles and access levels in the delegated work flow are described in Table 6-1.

 

CWNS Users have the Role of Federal, State, and/or Local.  The role Federal is limited to EPA headquarters staff, EPA Regional Coordinators, and the Contractor Review Team.  The role State role is assigned to State CWNS Coordinators.  They can assign the role State to Secondary State staff (e.g., State staff and State CWNS contractors).  State Users with appropriate State User Management (a.k.a. State User Managers) can assign the role Local to facilities, municipal governments, and other organizations interested in providing feedback on CWNS data.  The role Local can also be assigned to Secondary State staff.

 

For each Role, there are Access Levels.  The Access Level determines the type of activity that the user can perform within the DEP.  Activities range from viewing the data to submitting it for review to managing users.  Note: Users can and in many cases need to have more than one access level.  In addition, users can be granted access to all facilities/projects in the State, or they can be restricted to one or more facilities/projects.

State CWNS Coordinators are granted the access level of CWNS State User Management (a.k.a., State User Manager) after:

 

• Completing the CWNS user management training that provides instructions on the roles and responsibilities for delegation rights.  Training will be conducted at the Kick-off Meeting or by watching the CWNS training video “Administration Functions.”

 

• Signing a State User Manager Agreement, which outlines their responsibilities and the procedures used to manage delegation of rights. This agreement is maintained by the CWNS System Manager.  Users who do not attend the Kick-off meeting should fax the signed agreement (available in the CWNS 2008 Data Entry folder of the Portal Library) to the attention of Michael Plastino at 202-501-2397 no later than February 4, 2008.

 

The CWNS State User Management  access level provides State CWNS Coordinators the privilege to:

 

• Grant State User Manager privileges to other State Users

 

• Delegate CWNS DEP access to Secondary State and State Users  

 

• Limit Secondary State and State Users access to specific facilities and projects

 

• Modify access rights associated with Secondary State and State User accounts

 

• Deactivate, lock, or otherwise prevent the use of the accounts

 

In addition, State CWNS Coordinators are assigned all the other State access levels.

 

The State CWNS Coordinator must confirm the identity and credentials of all State Secondary or State Users who are given access to the DEP. Given that CWNS data is public domain, the State Coordinator simply confirms a user’s identity based on information either on hand or obtained from a known trusted contact, such as a letter using official letterhead (e.g., State, facility, municipal government) or a reference from a known trusted contact.
